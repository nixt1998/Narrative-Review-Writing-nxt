# Summary Table Template (v2.1)

Use this template for each table in Step 8.0.
Each table produces two outputs: a markdown version (07_tables.md) and
a standalone Word file (07_tables_standalone.docx).

---

## Design Principles

### Minimalist Design Rule (critical)
Every column in a table must provide information that cannot be as efficiently
presented in the prose. Remove any column that merely repeats body text content.
A table that duplicates the prose is worse than no table at all.

### Synthesis-Level Content
Every row must represent an integrated finding, not raw data enumeration.
The reader should learn something from the table that the prose alone does not convey.

### Informative Titles
Table titles must be conclusion statements, not generic labels.

Correct:
  Summary of NLRP3 Inhibitors and Their Mechanistic Targets in Preclinical Nephrotoxicity Models

Incorrect:
  Table of drugs and mechanisms

### Citation Rule for Tables
Table citations are exempt from the single-use-per-reference rule that applies to body text.
A reference that appears in a table may also appear in body text without counting as a repeat.
Reference numbering order follows the order of first appearance in body text; tables do not
affect this order.

## Suggested Column Structures

### Mechanism or Evidence Table

| Mechanism/Target | Key Evidence | Model System | Reference(s) | Gaps or Controversies |
|-----------------|-------------|-------------|-------------|----------------------|

Use this structure when comparing mechanistic findings across studies.

### Drug or Compound Comparison Table

| Drug or Compound | Dose/Concentration | Key Finding | Model | Reference(s) |
|-----------------|-------------------|------------|-------|-------------|

Use this structure when comparing pharmacological interventions.

### Clinical or Translational Table

| Study Design | Population | Outcome | Effect Size | Reference(s) |
|-------------|-----------|---------|------------|-------------|

Adapt columns to the specific content; do not force content into a generic template.

---

## Rules

1. Every row must be traceable to at least one verified reference [N]
2. Keep column count minimal: add a column only if it provides essential unique information
3. Avoid merging cells if it obscures the data structure
4. Table numbering is sequential in the order tables appear in the manuscript
5. The standalone .docx table file is formatted identically to the main manuscript:
   Times New Roman 10 pt, 1.5x spacing, justified. Table titles formatted as captions.
6. The .docx file is submitted independently -- it is NOT merged into the main manuscript

- [ ] Table numbered sequentially with other tables in the manuscript
