# Summary Table Template

Tables in narrative reviews must be **synthesis-level** -- each row represents an integrated finding, not a single-paper data point.

## Table Structure

```
Table [N]. [Title -- should be a core conclusion, not a descriptive label]

| Topic/Theme | Key Evidence | References | Gaps/Controversies |
|------------|-------------|-----------|-------------------|
| Integrated finding 1 | Synthesized evidence from multiple studies | [1,3,7] | What remains unclear |
| Integrated finding 2 | Synthesized evidence | [2,5] | Conflicting evidence |
| ... | ... | ... | ... |
```

## Rules

1. **Synthesis, not enumeration**: Each row summarizes across multiple studies, not a single paper
2. **Every row has a reference**: References column matches in-text citations and final reference list
3. **Consistent numbering**: Reference numbers [N] must be traceable to the verified literature list
4. **Gaps column**: Every row should identify what is NOT yet known (if nothing else, note the limitation)
5. **Table count**: Must match the user's parameter from Step 1.0

## Self-Check Before Output

- [ ] Every [N] in the table exists in the verified reference list
- [ ] No row is a single-paper summary
- [ ] Gaps/controversies identified for each topic
- [ ] Table count matches project parameter
