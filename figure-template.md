# Figure Instruction Template

For each figure, output the following structured instruction block. Designed to be handed to an AI drawing tool (e.g., BioRender, Adobe Illustrator, Inkscape) or a human scientific illustrator.

## Per-Figure Output Format

### Figure [N]: [One-line core meaning]

**Insertion location:** [Section X, after paragraph beginning with "..."] (matches framework marker)

**Core meaning (1 sentence):**
What this figure must convey to the reader at a glance.

**Elements to include:**
- [ ] Element 1 -- specific molecule/structure/process
- [ ] Element 2
- [ ] ...

**Visual hierarchy:**
- **Primary (most prominent):** [What the eye should go to first]
- **Secondary:** [Supporting context]
- **Background/tertiary:** [Labels, scale bars, subtle context]

**Logical flow:**
- Direction: left to right / top to bottom / circular
- Key transitions: [Step A]  [Step B]  [Step C]

**Color guidance:**
- Palette: [e.g., blue-orange diverging, single-hue sequential]
- Color-blind check: Use CVD-friendly palette (avoid red-green alone)
- Grayscale test: Foreground elements must remain distinguishable
- Key: [Color 1] = [Meaning], [Color 2] = [Meaning]

**Figure legend (ONE sentence -- core conclusion, NOT description):**
"Figure [N]. [Core conclusion sentence that captures the figure's take-home message.]"

**Abbreviations:**
| Abbreviation | Full Term |
|-------------|-----------|
| | |

## Quality Principles (from Dhillon 2022 Box 1)

- Keep content relatively simple
- Use arrows and numbers for multi-stage processes
- Add labels and scale bars as needed
- Use same font consistently within and across figures
- Leave ample white space around each element
- Avoid cramming too many components into one figure
- Use light, neutral background colors
- All panels neatly aligned
